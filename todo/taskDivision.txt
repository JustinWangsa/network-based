-  a login page and a page to purchase an account/"room"
- proposal, each user has their own account (their own username and password), then a company need to buy a room 
    - pros: each user dont need to remember the account/password for each company they enroll into, they just need to remember their own
    - cons: a breach on one account can potentially do dmg to multiple room


1. setiap user memiliki account masing-masing.  
2. user dapat membeli sebuah "room". Room belaku seperti sebuah toko. Di dalam room, manager dapat mengundang user lain, antara sebagai employee atau manager. 
3. ketika user berhasil login, akan tertera sederetan room yang mereka ikuti

keunggulan:
1. user hanya perlu mengingat 1 account-password. Tidak seperti sistem kita sekarang, dimana user perlu mengingat account-password untuk setiap tempat kerja mereka (ketika user ganti perkerjaan mereka harus mengingat account-password baru)
2. dengan sistem undangan, akun manager harus kebobolan, baru siapapun bisa memasuki suatu ruangan. Dibandingkan sistem saat ini, hanya perlu account-password (suatu text yang biasanya akan di sebar kepada coworker lewat text) kebobolan.
